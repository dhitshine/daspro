# Fundamental of Programming Archives

This repository contains the archive of course materials, laboratory exercises, and assignments for the Fundamental of Programming (Dasar Pemrograman) course at Institut Teknologi Sepuluh Nopember (ITS). All programming exercises and solutions in this archive are implemented in the C programming language.

## Repository Structure

The repository is organized into three main directories:

* [Modul](Modul/): Contains code examples, explanations, and exercises for each learning module.
* [Praktikum](Praktikum/): Contains laboratory problem sets, test cases, and solution codes.
* [Tugas](Tugas/): Contains course assignments and homework provided by the lecturer.

---

## Detailed Contents

### Modules (Modul)
This section contains structured guides and sample programs for learning C programming:
* **[Modul 0](Modul/Modul-0/)**: Introduction to C programming, basic syntax, and standard input/output operations.
* **[Modul 1](Modul/Modul-1/)**: Control flow mechanisms, conditional statements (`if-else`), logical operators, and basic mathematical operations.
* **[Modul 2](Modul/Modul-2/)**: Iterative structures (loops) and arrays, including exercises like prime number generation and array manipulation.
* **[Modul 3](Modul/Modul-3/)**: Functions, scope, recursion, and optimization techniques such as memoization.
* **[Modul 4](Modul/Modul-4/)**: Pointers, memory representation, and direct array manipulation using pointer arguments.

### Labs (Praktikum)
This section contains assignments and solutions from the scheduled programming labs conducted under competitive programming constraints:
* **[Praktikum 1](Praktikum/Praktikum-1/)**: Basic logic challenges, coordinate geometry, and conditional algorithm optimizations. (Solutions: [prak1.pdf](Praktikum/Praktikum-1/Solusi/prak1.pdf))
* **[Praktikum 2](Praktikum/Praktikum-2/)**: Intermediate logic challenges including bitwise shifting, iterative simulations, and numeric optimization. (Solutions: [prak2.pdf](Praktikum/Praktikum-2/Solusi/prak2.pdf))
* **[Praktikum 3](Praktikum/Praktikum-3/)**: Advanced recursion challenges, combination/permutation mathematics, and dynamic programming concepts. (Solutions: [prak3.pdf](Praktikum/Praktikum-3/Solusi/prak3.pdf))
* **[Praktikum 4](Praktikum/Praktikum-4/)**: Struct definitions, dynamic memory allocation (`malloc`/`calloc`/`free`), 2D arrays, and system simulation problems. (Solutions: [prak4.pdf](Praktikum/Praktikum-4/Solusi/prak4.pdf))
* **[Praktikum Final](Praktikum/Praktikum-Final/)**: A comprehensive collection of competitive programming problems covering all concepts taught throughout the semester.

### Assignments (Tugas)
This section contains course assignments and homework provided by the lecturer throughout the semester:
* **[Tugas 1](Tugas/Tugas-1/)**: Basic formula-based calculations, arithmetic operators, and user prompt configurations.
* **[Tugas 2](Tugas/Tugas-2/)**: Investigation of function flows and logical short-circuiting (`&&` and `||` evaluation).
* **[Tugas 3](Tugas/Tugas-3/)**: Implementation of conditional business logic (e.g., employee payroll calculations, overtime rates, and tax deductions).
* **[Tugas 4](Tugas/Tugas-4/)**: Modular application development with menu-driven flows and pointer parameters as function output arguments.
* **[Tugas 5](Tugas/Tugas-5/)**: File handling introduction (`fopen`, `fscanf`, `fprintf`), parallel arrays, and mathematical computations (e.g., polygon area calculation).
* **[Tugas 6](Tugas/Tugas-6/)**: Manipulating multi-dimensional character arrays (2D strings), string comparison functions (`strcmp`), and interactive user input loops.
* **[Tugas 7](Tugas/Tugas-7/)**: Recursive search algorithms (flood-fill / blob size calculation) on 2D grids with visited-state boundary tracking.
* **[Tugas 8](Tugas/Tugas-8/)**: Struct definitions, nested custom types, structure pointers, and modular data parsing/formatting.
* **[Tugas 9](Tugas/Tugas-9/)**: Advanced file manipulation, merging file streams using buffer size reading (`fgets`/`fputs`), and standard error handling with `stderr`.
